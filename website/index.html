Jnario

Executable Specifications for Java.

Jnario Specs

Write Facts about your Code

Jnario Specs describe facts about your code. Facts are examples describing the intended behavior of your code. A fact can be as simple as a single expression asserting a certain property. You can use should or => to describe the desired behavior of your objects. 

describe "The String 'Hello World'"{
  fact "Hello World".length should be 11
  fact "Hello World" should startWith "Hello"
  fact "Hello World" should endWith "World"
  fact "Hello World" should contain "Wor"	
}

Compile to Java - Run with JUnit

Jnario specs compile to plain Java classes, which are easy to understand and can be executed as normal JUnit tests. This makes Jnario really easy to integrate into existing Java projects.

Full Java interoperability

Jnario is completely type-safe and its type system resembles the one of Java. This makes it really easy to interact with Java objects . 

import java.util.ArrayList						

describe ArrayList{
  fact "adding increases size"{
    val list = new ArrayList<String>()
    list.size => 0
    list.add("something")
    list.size => 1
  }
}

More about Facts

You can also add a description to you facts to make the intention more clear. 

describe "Arithmetic"{
  facts "about addition"{
    1 + 1 => 2
    3 + 2 => 5
  }
}

As we are living in a mutable world, sometimes facts are only true in a certain context. Such contexts can be described by hierarchically structuring specs. 

describe Stack{
  val stack = new Stack
  context "empty"{
    fact stack.pop throws EmptyStackException
  }
  context "one element"{
    before stack.add("something")
    fact stack.pop => "something"
  }
}

A DSL designed for Testing

Jnario is domain-specific language with the syntax being specifially tailored to the purpose of writing tests. For example, specifying different sets of input parameters with the expected output values for an algorithm is simple with Jnario's table syntax. Tables in Jnario are type safe: the type of a column will be automatically inferred to the common super type of all cells in a column. 

describe "Arithmetic"{
  def additions{
        |  a  |  b  |  sum  |
        |  1  |  2  |   3   |
        |  4  |  5  |   9   |
        | 10  | 11  |  20   | 
        | 21  | 21  |  42   | 
  }
  fact additions.forEach[a + b => sum]
}

Selfexplaining Assertions

When an assertion fails, Jnario tries to give you as much information as possible about why it has failed. A failed assertion prints the values of all evaluated sub-expressions. This means you don’t need to debug your tests anymore to find the exact reason why an assertion failed. This works out of the box, without any additional constraints such as “your source files must reside in a specific folder”. 

java.lang.AssertionError: additions failed

        | a      | b      | sum     |
        | 1      | 2      | 3       | ✓
        | 4      | 5      | 9       | ✓
        | 10     | 11     | 20      | ✘     (1)
        | 21     | 21     | 42      | ✓

(1) Expected a + b => sum but
         a + b is 21
         a is 10
         b is 11
         sum is 20

Jnario Feature Specs

Cucumber-style Specs...

With Jnario you can create executable acceptance specifications that are easily readable and understandable to business users, developers and testers. Jnario uses a similar textual format to Cucumber. 

Feature: Addition

  In order to avoid silly mistakes
  As a math idiot
  I want to be told the sum of two numbers
  
  Scenario: Add two numbers
    Given I have entered "50" into the calculator
    And I have entered "70" into the calculator
    When I press "add"
    Then the result should be "120"

...with a Twist

There is one big difference between Jnario and Cucumber. You don’t need to create separate step definitions to make your steps executable. You can directly add the necessary code below your steps. 

Feature: addition
  ...
  Scenario: Add two numbers
    val calculator = new Calculator()
    Given I have entered "50" into the calculator
      calculator.enter(args.first)
    And I have entered "70" into the calculator
    When I press "add"
      calculator.press(args.first)
    Then the result should be "120" 
      calculator.result => args.first

Tool Support

You might think now that mixing code and text in your specs makes everything pretty unreadable. Actually, this is not a problem, as you can hide the code in the editor to improve the readability

Jnario Suite

Orchestrate Spec execution

A suite groups multiple specifications and executes them together. For example, create a single Jnario suite thats runs all your specifications in a project or create two different suites, one running all unit specifications and one running all acceptance specifications. 

package demo

import specs.*
import features.*

#My Suite

- "My Feature"
- "My Spec"

...and document

Jnario generates HTML reports for suites making them a great way to provide additional documentation. For example, you can create a suite as an overview page describing the overall architecture of an application, which then links to other specifications describing the different parts in more detail. 

package demo

import specs.*

#My Suite

This is the description of the suite. It is possible to use 
[Markdown Syntax](http://daringfireball.net/projects/markdown/syntax)
for **formatting** the text and adding images or links. 

##My Specs

This is a subsection in which we list all our unit specs:

- "My Spec": this is an example unit spec.
   with a multiline description.

##My Features

and here we list all our features using a regular expression:

- \.*Feature\

Eclipse Setup

Jnario requires Eclipse 3.5 or higher and a Java SDK 5 or higher. You can install Jnario runtime and tooling via the Eclipse Marketplace or using the following update sites:

Official releases:   http://www.jnario.org/updates/releases/

Stable Snapshots: http://www.jnario.org/updates/snapshot/

Maven Setup

Maven can be used for downloading the Jnario runtime library and for compiling the spec to Java.

Runtime Library:

<dependency>
  <groupId>org.jnario</groupId>
  <artifactId>org.jnario.lib.maven</artifactId>
  <version>0.1.0</version>
  <scope>test</scope>
</dependency>

Compiler:

<plugins>
  <plugin>
    <groupId>org.jnario</groupId>
    <artifactId>jnario-maven-plugin</artifactId>
    <version>0.1.0</version>
    <executions>
      <execution>
        <goals>
          <goal>testCompile</goal>
        </goals>
      </execution>
    </executions>
  </plugin>
</plugins>
