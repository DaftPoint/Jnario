Introduction

There is so much more to testing than just checking whether your code behaves as expected. Tests help you design your system. They specify the behavior of your classes and give examples on how to use an API. We think of tests as executables specifications guiding the design of your system.

Jnario is a framework for writing executable specifications for Java. It makes it easy to write acceptance tests for ... It provides all the goodness of Xtend while being specifically tailored to writing acceptance and unit tests.

Acceptance Specifications

 Feature: Addition

  In order to avoid silly mistakes
  As a math idiot
  I want to be told the sum of two numbers
  
  Scenario: Add two numbers
    Given I have entered "50" into the calculator
      var calculator = new Calculator()
      calculator.enter(args.first)
    And I have entered "70" into the calculator
      calculator.enter(args.first)
    When I press "add"
      calculator.press(args.first)
    Then the result should be "120" on the screen
}

Unit Specifications

package example

import java.util.Stack
import java.util.EmptyStackException

describe Stack{
  context "empty stack"{
    fact subject.size should be 0
    fact subject.pop throws EmptyStackException
  }
  context "one element"{ 
    before subject.add("element")
    fact subject.size should be 1
    fact "pop decreases size"{
      subject.pop
      subject.size => 0 
    }
    fact "pop removes last element"{
      subject.pop => "element"
      subject.contains("element") => true
    }  
  }   
}
